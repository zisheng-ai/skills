# Internationalization

## Supported Languages

Sites generated by this skill may target English, Spanish, Japanese, or Korean. The site must be language-aware from the first implementation pass — retrofitting i18n is expensive.

## HTML and Metadata

- Set `<html lang="en">`, `es`, `ja`, or `ko` as appropriate. This affects hyphenation, line breaking, and assistive technology.
- For multilingual sites where a book's language differs from the UI language, set `lang` on the `<article>` or the chapter `<div>` level.
- Use `<meta charset="UTF-8">` on every page.
- If content mixes languages in the same sentence or phrase, set `lang` on the inline `<span>` when practical.
- Dates: use `Intl.DateTimeFormat` with the correct locale, not manual string formatting.

## Font Stacks

Use system-first stacks. Never load a CJK web font for body text — system fonts are faster and render better on native OS than any web font fallback.

**English / Spanish (reader body):**
```css
font-family: ui-serif, Georgia, "Times New Roman", serif;
```

**Japanese (reader body):**
```css
font-family: "Hiragino Mincho ProN", "Hiragino Mincho Pro", "Yu Mincho", "YuMincho",
             "Noto Serif CJK JP", "Source Han Serif JP", serif;
```

**Korean (reader body):**
```css
font-family: "Apple SD Gothic Neo", "Noto Serif CJK KR", "Nanum Myeongjo",
             "Malgun Gothic", serif;
```

**All languages (UI, navigation, controls):**
```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI",
             "Hiragino Sans", "Yu Gothic UI", "Malgun Gothic", sans-serif;
```

Never use `NanumGothic`, `Dotum`, or legacy CJK UI fonts as the first choice — they look outdated on modern OS.

## Custom / Display Fonts

If the brief specifies a custom display font (for the site logo or book titles):
- Load only the display font, not a body font.
- Use `next/font` to self-host the font as a static asset — never load from an external CDN at runtime.
- Subset the font to the characters actually needed (Latin, specific CJK range).
- Keep the total font payload below 50KB for display-only use.

## Language-Specific Reading Notes

### English
- Line length: 60–76 characters per line (approximately `38em` at 17px).
- Hyphenation: `hyphens: auto` helps avoid ragged right edges for long words. Requires correct `lang="en"`.
- Chapter navigation labels: "Previous chapter", "Next chapter", "Chapter list", "Continue reading".

### Spanish
- Words and navigation labels are typically 20–30% longer than their English equivalents. Never use fixed-width buttons for navigation labels.
- Use `lang="es"` for correct hyphenation of compound words and accented characters.
- `hyphens: auto` is beneficial. Verify on sample long words: `independientemente`, `extraordinariamente`.
- Chapter navigation labels: "Capítulo anterior", "Capítulo siguiente", "Lista de capítulos", "Continuar leyendo".

### Japanese
- Default to horizontal writing (`writing-mode: horizontal-tb`) for H5 unless the user explicitly asks for vertical layout.
- If vertical (tategumi) mode is requested: `writing-mode: vertical-rl; text-orientation: mixed`. Test on mobile — vertical layout changes scroll direction (horizontal scroll).
- Do not apply `letter-spacing` to Japanese body text. It disrupts the natural rhythm of kanji and kana.
- Line height: 1.9–2.0 for horizontal; 1.5–1.8 for vertical.
- Chapter navigation labels: 「前の章」、「次の章」、「目次」、「続きを読む」.
- Punctuation: avoid orphaned 「」 or 。at the start of a line — CSS `hanging-punctuation: first allow-end` if supported.

### Korean
- Use generous line height: 1.9–2.1. Korean text needs more leading than CJK average.
- Avoid forced `text-justify: distribute` or `inter-character` justification — it creates uneven spacing between Hangul syllable blocks.
- Ensure Hangul word breaking does not leave isolated syllables. CSS `word-break: keep-all` prevents mid-word breaks for Korean.
- Chapter navigation labels: 「이전 장」, 「다음 장」, 「목차」, 「계속 읽기」.

## Locale-Aware Formatting

```js
// Dates
new Intl.DateTimeFormat('ja', { year: 'numeric', month: 'long', day: 'numeric' })
  .format(new Date('2026-06-28'));
// → "2026年6月28日"

// Word count
new Intl.NumberFormat('en').format(12345);
// → "12,345"

// Always use the book's language for in-book formatting, not the UI language.
```

## Layout Resilience

- Use flexible grids and flexbox, not fixed pixel widths on text containers.
- Navigation labels must wrap gracefully. Never truncate UI labels — expand the container.
- Avoid icon-only controls unless accessible `aria-label` is present and a visible label is impractical.
- Test with both the longest expected title (`La ciudad que olvidó sus nombres`) and the shortest (`雨`).
- Book cards must handle 1-character CJK titles and 60-character English titles without overflow or clipping.

## Translation Key Naming (If Implementing i18n Library)

```js
// Prefer semantic keys over positional strings
t('reader.nav.previousChapter')   // not t('btn.prev')
t('reader.nav.nextChapter')
t('reader.settings.fontSize')
t('reader.settings.theme.light')
t('reader.settings.theme.sepia')
t('reader.settings.theme.dark')
t('book.status.ongoing')
t('book.status.completed')
t('book.status.hiatus')
t('home.continueReading')
t('home.startReading')
t('error.chapterNotFound')
t('error.loadFailed')
```

For prototypes without an i18n library, a simple object map is sufficient:

```js
const i18n = {
  en: { previousChapter: 'Previous chapter', nextChapter: 'Next chapter' },
  ja: { previousChapter: '前の章', nextChapter: '次の章' },
  ko: { previousChapter: '이전 장', nextChapter: '다음 장' },
  es: { previousChapter: 'Capítulo anterior', nextChapter: 'Capítulo siguiente' },
};
```
